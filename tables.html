Tables

Tables and grids are important UI components for representing complex data and dashboards. Good data tables allow users to scan, analyze, compare, filter, sort, and manipulate information to derive insights and commit actions. This component is especially important for power users.

  • Use Cases

Default Tables

This variant of tables is used when no actions are required at the table row level. Columns indicating status, priority, etc. with icons are generally the first columns on the table.

Tables with Actions

This variant of tables allows users to take quick actions at the table row level. Checkboxes are provided for the user to select one, multiple, or all rows in the view and perform bulk actions. The actions column positioned as the first column lets users find and perform actions quickly (versus having the column in any other position). Similar to lists, a maximum of two primary actions can be exposed as text buttons, and the rest are tucked inside the overflow menu.
