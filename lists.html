Lists

Lists are a collection of similar data types represented as a continuous column of rows. They are sorted in logical ways that make content easy to scan, such as alphabetical, numerical, chronological, or by user preference. The content within the list can take different forms, depending on the needs of the list.

List Items in Articles

Use this variant of lists without a unique identifier generally for informal lists that are not a part of the core functionality.

Without Actions

In this variant, the list item contains a primary data item, a secondary description, and a tertiary timestamp. The list is used to represent a collection of data that can be further drilled down, but no row-level actions can be taken in this kind of list.

With Actions

In cases where users are expected to frequently interact with the list items and transform them (edit, delete, or other actions), quick row-level actions are provided. The first two primary and most frequently used actions are presented as text buttons, and all successive actions are tucked inside an overflow menu located towards the right. Note: in this variant, the list itself cannot be clicked, hence the buttons must be used to present drill-down options.

List Items in with a Unique Identifier

A unique identifier is used on lists to further emphasize the primary item on the list. This identifier also serves as an element for the user to drill down for further information and leads to a dedicated sub-page. The identifier should not be used for any other action or utility (like directly taking the user into an edit view, etc.). Use this variant of lists for formal lists that are a part of the core functionality.
