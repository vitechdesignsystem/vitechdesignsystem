Widgets

Widgets are the foundations of all features in our design system, and hence our products. They are placed directly on the canvas and contain many elements from our library like lists, buttons, pills, tables, etc., that come together to create a unique feature.

  • Design Guidelines
  • Use Cases

Default

Widget headers contain the widget name to inform the user what the widget is about. The header also provides up to two primary actions as text buttons, with the rest tucked inside the overflow menu. When appropriate, widgets also provide search, sort, and filtering capabilities.

Pills and button groups are sometimes used below the header to switch between the multiple available views.

The widget body is usually a list, table, plain text, or an amalgamation of these to form a complex element (like secure messages, etc.).
