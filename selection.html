Selection

Selection controls allow users to complete tasks that involve making choices such as selecting options or switching settings on or off. Selection controls are found on screens that ask users to make decisions or declare preferences such as settings or dialogs.

  • Design Guidelines
  • Use Cases

Checkboxes

Checkboxes are used when there are lists of options, and the user may select any number of choices, including zero, one, or several. In other words, each checkbox is independent of all other checkboxes in the list, so checking one box doesn't uncheck the others. A stand-alone checkbox is used for a single option that the user can turn on or off.

Radio Buttons

Radio buttons are used when there is a list of two or more options that are mutually exclusive, and the user must select exactly one choice. In other words, clicking a non-selected radio button will deselect the button previously selected in the list. It is important to note that a user is expected to select at least one option in a radio button.

Switches

Toggle switches are digital on/off switches. They prompt users to choose between two mutually exclusive options and always have a default value. They are best used for changing the state of system functionalities and preferences. Toggles may replace two radio buttons or a single checkbox to allow users to choose between two opposing states.

Defaults and Grouping

Radio buttons must always offer a default selection. By definition, radio buttons always have exactly one option selected, and you, therefore, shouldn't display them without a default selection. If users might need to refrain from making a selection, you should provide a radio button for this choice, such as one labeled "None." Offering users an explicit, neutral option to click is better than requiring the implicit act of not selecting from the list.

Don't
  • Avoid ungrouped selection controls.
  • Radio buttons must always have a default selection.
Recommended
  • Group options under meaningful headings.
  • A selected option can toggle related fields.

Usage and Positioning

Standard visual representations must be adhered to. A checkbox should be a small square that has a checkmark when selected.

When grouped, checkboxes cannot have interlinked dependencies, i.e. selecting one cannot affect the other checkboxes in the group. However, you may limit the max number of choices (to two or more). As it is easier to visually scan individual items when they are stacked one below the other, exercise caution while laying out checkboxes horizontally.

Don't
  • Don't use checkboxes to allow only 1 selection.
  • Checkboxes cannot have interlinked dependencies.
Caution
  • Stacking checkboxes horizontally can be hard to read, especially when there are a lot of options.
