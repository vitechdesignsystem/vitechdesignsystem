Input Fields

Input fields are used to capture user input on forms, modals and wizards among other places. This is done using text input fields and textareas that allow users to enter text into a field. We use a few appropriate variants at different places throughout our apps like input fields with help texts, search boxes, outlined text inputs, etc.

  • Design Guidelines
  • Use Cases

Regular Text Inputs

Obtain user inputs within forms and modals. Use a regular text input when the number of words to be input by the user is few, generally within one line. As these input boxes do not expand in width or height, using them for multi-line texts would not be ideal.

Small Text Inputs

Small text inputs are used as in-situ inputs without labels. They are found on interfaces that generally don't change, with the users being familiar with the interfaces even outside of Vitech's apps. A search bar is an example of this, something that users can recognize by immediately looking at, even without a label for the input.

Textareas

Textareas are long-form inputs where users are generally expected to input multi-line texts. These input boxes grow to a few lines (as configured) and might even show a scroll bar for a user to read all the typed content. A control is provided on the bottom-right side of the input box for the user to manually adjust the size of the textarea.

Error Handling

The error handling mechanism built into the input fields can be a great way to inform the user about specific, field-level errors. Presenting errors in this format allows the user to find the input fields with missing or incorrect inputs quicker.

Recommended
  • Use concise text to pinpoint the error(s).
  • Validate as the user fills the form, or on submit.
Caution
  • Small text makes multi-line messages harder to read.
  • May not be appropriate for smaller devices.

Uniform Field Sizes

Field sizes must be uniform and be aligned appropriately unless they follow a multi-column layout. It ensures a clean, consistent, and professional-looking interface.

Recommended
  • Although the modal here is full-width, keeping the field lengths shorter ensures better readability and lesser lines going across the screen.

Grouped Inputs

Related input fields can be grouped together on the same line with a multi-column layout. Note that these should be relatively simple fields (like a date field with day, month, year, etc.). Grouping allows users to perceive individual fields as being a part of a whole.

Recommended
  • As the marital date field is related to the user being married, they are grouped on the same line.
  • This field can also be hidden when not needed (in the case when the user is not married).
